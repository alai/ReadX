🎯 识别策略分析

可行的识别方法：

1. data-testid 属性定位

✅ 高可行性

article[data-testid="tweet"] div[data-testid="tweetText"] div[data-testid="User-Name"]

优势：这些属性相对稳定，是为了测试而设计的

2. 语义化HTML标签

✅ 高可行性

article[role="article"] // 主贴通常被包装在article标签中

优势：符合Web语义化标准，相对稳定

3. aria-label 属性

✅ 中等可行性

article[aria-labelledby] // 可用于无障碍访问的标识

优势：为无障碍访问而设计，变动较少

🚫 不可行的方法

❌ 不可行

💡 推荐的实现策略

优先级排序的选择器策略：

// 1. 最优先：使用 data-testid const tweetSelector = 'article[data-testid="tweet"]'; const tweetTextSelector = 'div[data-testid="tweetText"]'; // 2. 备选：语义化标签 const fallbackTweetSelector = 'article[role="article"]'; // 3. 最后备选：结构特征 const structuralSelector = 'article[aria-labelledby]'; // 实现逻辑 function findMainTweet() { // 按优先级尝试 let tweet = document.querySelector(tweetSelector); if (!tweet) { tweet = document.querySelector(fallbackTweetSelector); } if (!tweet) { tweet = document.querySelector(structuralSelector); } return tweet; }
